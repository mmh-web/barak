Skills — Teaching Barak New Abilities

A Skill is a written instruction set that permanently teaches Barak how to handle a specific type of task. Write it once, and he follows it every time — no reminders needed.

01

Why Skills Matter

Without Skills

  • Every conversation, you re-explain your preferences
  • Source lists, formatting rules, and constraints get forgotten
  • The agent drifts from what you actually want
  • Tribal knowledge stays in your head, not in the system

With Skills

  • Write instructions once as a structured document
  • The agent reads them automatically when the task matches
  • Consistency every single time — no drift
  • Like an employee handbook entry that's actually followed

Skills turn tribal knowledge into executable instructions. They're the bridge between "I told you this before" and "I always do it this way."

02

How Skills Work

1

A Skill is a markdown file

Each Skill lives as a SKILL.md file stored in the agent's workspace. Plain text, human-readable, version-controlled.

2

It has a description that tells the agent WHEN to use it

The description acts as a trigger. When an incoming task matches, the agent knows to load that Skill automatically.

3

The agent reads it and follows it exactly

No interpretation, no drift. The Skill file is the source of truth — the agent treats it as law.

4

Skills can reference other files

Source lists, templates, API configurations, buyer maps — a Skill can pull in any supporting document it needs.

5

Update them anytime

Skills are version-controlled files. Change the instructions, and the agent follows the new version immediately.

Real Examples from Barak's Current Skills

Morning Briefing Skill

Custom

Searches for hiring fraud news every morning and delivers a curated briefing to Menachem at 6:30 AM PT.

"Max 5-7 bullets. Real journalism only — Reuters, AP, Bloomberg, WSJ tier. No vendor blogs. If it's a quiet news day, say so."

Without this Skill, Barak would need to be told the source rules and format every single time.

GitHub Workflow Skill

Built-in

Manages code via GitHub — pull requests, issues, commits, and CI checks. The guardrails that keep production safe.

"Never push directly to main. Create feature branches. Open PRs for review."

Prevents the agent from accidentally pushing bad code to production.

Weather Skill

Built-in

Gets weather forecasts from free APIs — a simple example of how a Skill packages a capability.

"Uses wttr.in or Open-Meteo. No API key needed."

Simple example of how a Skill wraps an external service into a reliable, repeatable capability.

Built-in

"Handles OAuth flows, permission checks, and re-auth automatically. Sends auth buttons when permissions are missing."

Complex integration made simple through structured instructions.

04

Creating New Skills

This is where it gets exciting. Here are examples of Skills that Menachem could teach Barak — each one turns a recurring need into an automated, consistent process.

Content Calendar Skill

"Every Monday, scan the week's briefings and suggest 2-3 content angles for the Clarity blog. Format as: headline, 2-sentence pitch, target keyword, source briefings."

Competitor Monitoring Skill

"Every week, check what Reality Defender, Pindrop, and other competitors have published. Flag any new product announcements, funding rounds, or partnerships."

LinkedIn Comment Drafting Skill

"When given a LinkedIn post URL, analyze the post, identify which buyer lane it fits, and draft a comment in Michael's voice. 2-4 sentences. Add perspective, don't pitch."

Meeting Prep Skill

"Before any meeting with a buyer target, pull their latest news, check the buyer map score, review past interactions from relationships.md, and create a one-page brief."

How to Create a Skill

Just tell Barak: "I want to teach you a new skill for [X]." Describe what you want done, when, how, and any rules. Barak writes the SKILL.md file, and from then on, that task is handled consistently every time.

You can refine it by just talking — "make it shorter," "add this source," "change the format." The Skill file updates, and the new behavior takes effect immediately.

05

The Power of Skills

  • Skills compound
    Each one you add makes the agent more capable and more consistent. Over time, Barak becomes deeply specialized in your workflows.
  • Skills are transparent
    You can read exactly what instructions the agent follows. No black box — every rule is visible and auditable.
  • Skills are editable
    Change them anytime by just talking. Say "update the briefing skill to include SEC filings" and it's done.
  • Skills survive context
    Unlike conversation instructions that get lost in compaction, Skills are files that persist forever. They never fade.
  • Skills scale
    What works for one task pattern can be templated for others. One good Skill becomes a blueprint for many.
